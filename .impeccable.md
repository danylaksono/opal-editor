## Design Context

### Users

Researchers, scientists, academics, and other long-form writers using Opal to create and compile LaTeX projects locally. They need to understand the editor quickly, find specialised features when required, and remain in control of optional AI-assisted workflows.

### Brand Personality

Calm, careful, and author-controlled. Opal should feel like a focused scholarly tool: capable without being noisy, reassuring without being patronising, and clear about what is local, optional, or user-approved.

### Aesthetic Direction

A refined editorial interface that continues Opal's existing warm-paper and deep-ink visual language. Documentation should prioritise readable typography, strong information hierarchy, restrained colour, and useful product imagery. Both light and dark application themes remain supported; the public documentation itself should default to a bright, comfortable reading surface.

### Design Principles

- Make the common path obvious, then progressively disclose advanced workflows.
- Keep the author in control, especially around AI, interactivity, and external services.
- Prefer precise, searchable guidance over long onboarding sequences.
- Show the real interface and use academic examples rather than generic mock content.
- Preserve calm visual rhythm, accessible contrast, keyboard navigation, and readable line lengths.
